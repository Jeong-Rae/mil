# Audit Rules

개발 시 아래 원칙을 따른다.

## 최소 구현

- 현재 명세에 있는 기능만 구현한다.
- 미래 확장, 재사용, 설정 가능성을 미리 넣지 않는다.
- 고정값은 고정값으로 둔다.
- 호출자와 경로가 하나면 그 전제에 맞춰 단순하게 만든다.

## 최소 검증

- 현재 고정된 클라이언트와 요청 흐름에서 필요한 검증만 한다.
- 과도한 오류 처리, fallback, 복구 흐름을 넣지 않는다.
- 단, 한 요청이나 한 연결의 실패가 전체를 죽이지 않도록 하는 얇은 보호는 둔다.

## 구현 기준

- 한 번만 쓰는 값은 불필요하게 변수나 옵션으로 분리하지 않는다.
- 인라인 코드가 너무 길어질 때만 작은 유틸리티로 분리한다.
- 추상화는 현재 코드 길이와 복잡도를 줄일 때만 도입한다.

## 빠른 판단 기준

- 지금 명세에 꼭 필요한가?
- 지금 고정된 사용 경로에서 실제로 필요한 검증인가?
- 유틸리티 분리가 코드 길이와 이해 비용을 실제로 줄이는가?

셋 중 하나라도 아니면 넣지 않는다.
