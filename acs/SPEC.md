# AccessControll Service Spec

## 1. 개요

AccessControll Service(이하 ACS)는 군부대 장병의 입영 및 퇴영을 관리하기 위한 로컬 기반 프로그램이다.

1단계 구현은 **네트워크가 연결되지 않은 Local 머신 환경**을 기준으로 하며,  
바코드 스캐너로부터 입력되는 **문자열 시퀀스**를 처리하여 입영/퇴영 로그를 기록한다.

향후 동일 구조를 바탕으로 네트워크 연동 기능을 확장할 수 있도록 설계하되,  
현재 단계에서는 로컬 파일 입출력과 단일 프로세스 실행만을 대상으로 한다.

---

## 2. 개발 기준

- 구현 언어: **PowerShell**
- 실행 환경: **Windows PowerShell 5.x**
- 엔트리 파일: `ACS.ps1`
- 문서 기준: 본 Spec은 **1단계 로컬 서버 코드**를 기준으로 한다.
- 구현 원칙:
  - 과도한 validation 지양
  - 과도한 argument parsing 지양
  - 실사용 중심의 단순한 흐름 우선
  - 이후 네트워크 확장이 가능하도록 책임 분리 고려

---

## 3. 목적

ACS의 목적은 다음과 같다.

- 장병의 입영/퇴영을 바코드 입력으로 처리한다.
- 바코드로부터 장병 식별 정보를 복원한다.
- 해당 결과를 로그로 남긴다.
- 중복 스캔을 단시간 내 무시하여 오입력을 방지한다.

---

## 4. 시스템 범위

### 4.1 1단계 범위

1단계에서 ACS는 다음 기능만 제공한다.

- 로컬 파일 `./list.json` 에서 장병 목록을 읽음
- 서버 실행 시 전달받은 장소(arg)를 현재 실행 장소로 사용
- 콘솔 또는 표준입력을 통해 바코드 문자열 시퀀스를 수신
- 바코드 종류(입영/퇴영)를 판별
- 바코드 시퀀스를 decode 하여 군번을 식별
- 장병 정보를 조회
- 중복 입력 여부를 검사
- 결과를 CSV 로그로 저장

### 4.2 범위 제외

1단계에서 아래 항목은 구현 범위에서 제외한다.

- 네트워크 통신
- 중앙 서버 동기화
- 실시간 다중 클라이언트 처리
- 복잡한 사용자 권한 체계
- 고급 예외 복구
- DB 연동
- GUI

## 5. 용어 정의

### 5.1 ACS
AccessControll Service를 의미한다.

### 5.2 장병 구분 코드 / 군번
장병을 식별하는 기본 키이다.  
형식 예시는 다음과 같다.

- `a00-0000000`

단, `00` 영역의 자릿수 및 의미는 직위 및 구분에 따라 달라질 수 있으므로,  
실제 구현에서는 **고정 포맷 완전 검증보다 문자열 식별값**으로 취급한다.

### 5.3 입영 바코드
입영 처리용으로 발급된 바코드 문자열이다.

### 5.4 퇴영 바코드
퇴영 처리용으로 발급된 바코드 문자열이다.

### 5.5 시리얼넘버
각 바코드 문자열에서 중복 판정에 사용되는 고유 식별값이다.  
동일 시리얼넘버는 **입력 후 15초 이내 재입력 시 무시**된다.

## 6. 입력 데이터

## 6.1 장병 목록 파일

장병 정보는 `./list.json` 에서 로드한다.

### 6.1.1 파일 위치
- 상대경로: `./list.json`

### 6.1.2 기본 역할
- 군번 기준 장병 정보 조회

### 6.1.3 예시 구조

```json
[
  {
    "id": "a01-1234567",
    "name": "홍길동",
    "rank": "병장",
    "unit": "A중대"
  },
  {
    "id": "b12-7654321",
    "name": "김철수",
    "rank": "상병",
    "unit": "B중대"
  }

```

### 6.1.4 필수 사용 필드

1단계에서 실제 처리에 필수인 값은 아래와 같다.

* `id` : 군번
* `name` : 이름

그 외 필드는 보존 가능하나, 1단계 핵심 로직에서는 필수로 사용하지 않는다.

## 7. 실행 인자

ACS는 서버 실행 시 **장소(place)** 를 인자로 입력받는다.

예시:

```powershell
powershell -ExecutionPolicy Bypass -File .\ACS.ps1 gate-1
```

또는

```powershell
.\ACS.ps1 gate-1
```

여기서 `gate-1` 이 장소 값이다.

스크립트 파일 실행이 제한된 환경에서는, `ACS.ps1` 내용을 PowerShell 세션에 복사 붙여넣기한 뒤 아래처럼 함수 엔트리로 실행할 수 있어야 한다.

```powershell
Start-ACS -Place gate-1
```

현재 작업 디렉토리가 `acs` 디렉토리가 아닐 경우에는 앱 루트를 명시할 수 있어야 한다.

```powershell
Start-ACS -Place gate-1 -AppRoot C:\acs
```

### 7.1 장소 값의 용도

로그 저장 시 결과 엔트리에 포함된다.

### 7.2 장소 값 처리 원칙

* 단순 문자열로 취급한다.
* 과도한 인자 검증은 하지 않는다.
* 값이 없을 경우의 정책은 구현 단에서 단순 처리한다.

  * 예: `"unknown"` 또는 빈 문자열 허용

### 7.3 실행 방식 지원

구현은 아래 두 실행 방식을 모두 지원해야 한다.

* 스크립트 파일 직접 실행
* 스크립트 본문 복사 붙여넣기 후 함수 호출 실행

## 8. 바코드 처리 규칙

## 8.1 바코드 종류

바코드는 아래 두 종류가 존재한다.

* 입영 바코드
* 퇴영 바코드

두 바코드는 **서로 구분 가능한 형태**여야 한다.

## 8.2 바코드 내용

바코드는 군번을 기초로 한 **결정론적 알고리즘**에 의해 encode/decode 된다.

즉:

* 같은 군번은 항상 같은 규칙으로 encode 가능해야 한다.
* 입력된 바코드는 decode 시 항상 동일 군번 및 관련 정보로 복원 가능해야 한다.

## 8.3 알고리즘 상세화 원칙

본 1단계 Spec에서는 encode/decode 알고리즘의 수학적 상세는 고정하지 않는다.
다만 구현은 아래 계약을 만족해야 한다.

### 8.3.1 요구사항

* 입력 문자열만으로 바코드 종류(입영/퇴영)를 식별할 수 있어야 한다.
* 입력 문자열로부터 최소한 아래 값을 복원할 수 있어야 한다.

  * 종류(entry/exit)
  * 군번(id)
  * 시리얼넘버(serial)
* 동일 입력은 항상 동일 결과를 반환해야 한다.

### 8.3.2 권장 형태

예를 들어 다음과 같은 내부 구조를 가질 수 있다.

* prefix : 입영/퇴영 구분
* payload : 군번 기반 인코딩 값
* serial : 중복 판정용 시리얼

단, 실제 문자열 포맷은 구현에 위임한다.

---

## 9. 중복 입력 처리

동일한 시리얼넘버가 입력된 후 **15초 이내**에 다시 입력되면, 해당 입력은 무시한다.

## 9.1 판정 기준

* 키: `serial`
* 비교 범위: 최근 처리된 입력 이력
* 시간 기준: 처리 시각 기준 15초

## 9.2 동작

* 15초 이내 재입력: 무시
* 15초 초과 후 재입력: 정상 처리 가능

## 9.3 목적

* 스캐너의 연속 입력
* 손떨림 또는 중복 스캔
* 의도치 않은 재전송

등을 완화한다.

---

## 10. 처리 흐름

ACS의 기본 처리 흐름은 아래와 같다.

1. 서버 시작
2. 실행 인자에서 장소(place) 확보
3. `./list.json` 로드
4. 입력 대기
5. 바코드 문자열 수신
6. 바코드 decode
7. 종류(entry/exit), 군번(id), 시리얼(serial) 추출
8. 최근 15초 이내 동일 serial 존재 여부 확인
9. 중복이면 무시
10. 중복이 아니면 `list.json` 에서 군번으로 장병 조회
11. 결과 엔트리 생성
12. CSV 파일에 append 저장
13. 다시 입력 대기

---

## 11. 로그 저장

입퇴영 결과는 CSV 파일로 저장한다.

## 11.1 저장 형식

각 엔트리는 아래 필드를 포함한다.

* 시간
* 구분
* 장소
* 군번
* 이름

## 11.2 CSV 컬럼 정의

| 컬럼명   | 설명           |
| ----- | ------------ |
| time  | 처리 시각        |
| type  | 입영 또는 퇴영     |
| place | 실행 인자로 받은 장소 |
| id    | 군번           |
| name  | 이름           |

## 11.3 예시

```csv
time,type,place,id,name
2026-03-06T08:12:11,entry,gate-1,a01-1234567,홍길동
2026-03-06T17:41:53,exit,gate-1,b12-7654321,김철수
```

## 11.4 저장 방식

* 파일이 없으면 생성
* 파일이 있으면 append
* 인코딩은 PowerShell 5 호환성을 고려하여 단순한 방식 사용
* 지나친 파일 회전(log rotation) 기능은 1단계 범위에서 제외

## 11.5 파일명

1단계에서는 단순한 정책을 사용한다. 예:

* `./logs/access-log.csv`

또는 날짜 분리 방식:

* `./logs/access-log-YYYYMMDD.csv`

구체 파일명은 구현 선택 사항이나, **CSV append가 쉬운 구조**를 유지한다.

## 12. 장병 조회 규칙

decode 결과에서 얻은 군번을 기준으로 `./list.json` 에서 장병을 조회한다.

## 12.1 조회 키

* `id`

## 12.2 조회 결과

* 일치 장병 존재: 이름 포함하여 로그 저장
* 일치 장병 없음: 정책 선택 가능

### 12.2.1 권장 정책

1단계에서는 아래 둘 중 하나를 사용한다.

* 이름을 `"UNKNOWN"` 으로 기록
* 해당 입력을 무시하고 별도 오류 메시지만 출력

단, 구현 복잡도를 낮추기 위해 한 가지 정책으로 고정하는 것이 바람직하다.

권장안:

* **로그는 남기되 이름은 `UNKNOWN` 처리**

이 방식이 추후 데이터 점검에 유리하다.

## 13. 콘솔 동작 방식

1단계 ACS는 로컬 머신에서 실행되며, 바코드 스캐너가 일반적으로 키보드 입력처럼 동작하는 환경을 상정한다.

따라서 입력은 아래 방식 중 하나로 처리할 수 있다.

* `Read-Host` 기반 한 줄 입력
* 표준입력 루프
* 콘솔 입력 버퍼 기반 처리

1단계 구현에서는 **단순한 줄 단위 입력**을 권장한다.

즉, 스캐너가 문자열 입력 후 Enter를 보내는 형태를 가정한다.

### 14.4 `Write-AccessLog`

### 14.5 `Process-Barcode`

## 15.1 저장 대상

* `serial`
* 마지막 처리 시각

## 18. 최소 구현 계약

1단계 ACS가 충족해야 하는 최소 구현 계약은 아래와 같다.

1. `./list.json` 에서 장병 정보를 읽을 수 있어야 한다.
2. 서버 실행 시 장소 문자열을 인자로 받을 수 있어야 한다.
3. 바코드 문자열 입력을 받을 수 있어야 한다.
4. 입력 문자열을 decode 하여 입영/퇴영, 군번, 시리얼을 얻을 수 있어야 한다.
5. 동일 시리얼의 15초 내 재입력은 무시해야 한다.
6. 처리 결과를 CSV에 기록해야 한다.
7. CSV 엔트리는 시간, 구분, 장소, 군번, 이름을 포함해야 한다.

## 19. 예시 시나리오

### 19.1 정상 입영

* 실행: `.\ACS.ps1 gate-1`
* 또는: `Start-ACS -Place gate-1`
* 입력: 입영 바코드 문자열
* decode 결과:

  * type = `entry`
  * id = `a01-1234567`
  * serial = `SN0001`
* `list.json` 조회 결과 이름 = `홍길동`
* CSV 기록:

```csv
2026-03-06T08:12:11,entry,gate-1,a01-1234567,홍길동
```

### 19.2 15초 내 중복 스캔

* 동일 `SN0001` 재입력
* 이전 처리 후 15초 이내
* 결과:

  * 무시
  * CSV 기록 없음

### 19.3 정상 퇴영

* 입력: 퇴영 바코드 문자열
* decode 결과:

  * type = `exit`
  * id = `a01-1234567`
  * serial = `SN0099`
* CSV 기록:

```csv
2026-03-06T17:41:53,exit,gate-1,a01-1234567,홍길동
```
