# eX-chat Spec

## 1. 개요

eX-chat은 내부망에서 임시로 사용하는 단일 채팅방 프로그램이다.

- 테스트성 코드이며 장기 유지보수를 목표로 하지 않는다.
- PowerShell 스크립트와 정적 웹 파일만으로 구성한다.
- 가장 중요한 목표는 "기본 채팅이 동작하고 쉽게 crash 나지 않는 것"이다.
- 보안, 인증, 영속 저장, 복잡한 검증은 우선순위가 낮다.

---

## 2. 기술 기준

- 서버: PowerShell 스크립트 (`server.ps1`)
- 프론트엔드: `index.html`, `script.js`, `style.css`
- 포트: `8888`
- 바인드: `+:8888`
- 통신:
  - HTTP: 정적 파일 제공
  - WebSocket: 채팅 메시지 브로드캐스트

---

## 3. 시스템 범위

### 3.1 포함 범위

- 단일 채팅방 1개 제공
- 모든 사용자는 접속 즉시 같은 방에 참여
- 서버는 정적 파일을 HTTP로 제공
- 서버는 WebSocket 연결을 수락
- 서버는 받은 메시지를 현재 접속자 전체에 브로드캐스트
- 서버는 접속/종료 시스템 메시지를 브로드캐스트

### 3.2 제외 범위

- 메시지 저장
- DB 사용
- 파일 로그 저장
- 인증/인가
- 닉네임 입력
- 다중 채팅방
- 파일 업로드
- 복잡한 메시지 검증
- 외부망 공개 운영

---

## 4. 구조

프로젝트 파일은 아래와 같다.

- `server.ps1`
- `index.html`
- `script.js`
- `style.css`

`server.ps1`가 HTTP 서버와 WebSocket 서버를 함께 제공한다.

브라우저 사용자는 아래 주소로 접속한다.

```text
http://<host>:8888/
```

WebSocket 엔드포인트는 아래와 같다.

```text
ws://<host>:8888/ws
```

---

## 5. 서버 책임

서버는 최대한 얇게 구현한다.

### 5.1 서버가 담당하는 것

- `/`, `/index.html`, `/script.js`, `/style.css` 응답
- `/ws` WebSocket 업그레이드 처리
- 클라이언트 메시지 수신
- 전체 접속자 대상 브로드캐스트
- 접속/종료 시스템 메시지 전송
- 요청 IP 기반 접속자 명칭 계산

### 5.2 서버가 담당하지 않는 것

- 빈 문자열 입력 방지
- timestamp 생성/표시
- 입력창 UX 제어
- 메시지 렌더링
- 메시지 목록 화면 관리

이런 작업은 가능한 한 `script.js`에서 처리한다.

---

## 6. 사용자 식별 규칙

접속자 명칭은 요청 IPv4 주소 `A.B.C.D` 중 마지막 두 옥텟 `C.D`를 사용한다.

예시:

- `192.168.10.23` -> `10.23`
- `10.0.7.41` -> `7.41`

IPv6는 이번 범위에 포함하지 않는다. IPv6 또는 비정상 주소가 들어오면 구현은 단순한 대체 문자열로 처리해도 된다.

---

## 7. 메시지 규칙

### 7.1 클라이언트 -> 서버

- 클라이언트는 일반 텍스트 메시지를 WebSocket으로 전송한다.
- 별도 REST API는 사용하지 않는다.

### 7.2 서버 -> 클라이언트

서버는 JSON 텍스트 프레임으로 아래 두 종류의 메시지를 브로드캐스트한다.

#### 일반 채팅 메시지

```json
{
  "type": "chat",
  "sender": "10.23",
  "text": "hello"
}
```

#### 시스템 메시지

```json
{
  "type": "system",
  "text": "10.23 joined"
}
```

서버 timestamp는 필수가 아니다. 화면 표시용 시각은 클라이언트가 렌더링 시점에 생성한다.

---

## 8. 프론트엔드 책임

프론트엔드의 대부분 동작은 `script.js`에서 처리한다.

### 8.1 포함 책임

- WebSocket 연결 생성
- 연결 상태 표시
- 입력값 읽기
- 빈 문자열 또는 공백-only 문자열 전송 방지
- Enter 키 전송
- 메시지 목록 렌더링
- 시스템 메시지와 일반 메시지 구분 렌더링
- timestamp 생성 및 표시

### 8.2 표시 규칙

- 일반 메시지는 `sender`, `text`, `timestamp`를 함께 보여준다.
- 시스템 메시지는 별도 스타일로 보여준다.
- timestamp는 브라우저의 현재 시각 기준으로 생성한다.

---

## 9. 예외 처리 원칙

- 한 연결의 오류가 전체 서버 종료로 이어지지 않아야 한다.
- 잘못된 프레임, 끊긴 연결, 전송 실패는 해당 연결만 정리하면 된다.
- 메시지 저장이 없으므로 서버 재시작 시 모든 상태는 초기화된다.
- 복잡한 재시도, 복구, 검증은 요구하지 않는다.

---

## 10. 실행 방식

예시:

```powershell
pwsh -File .\server.ps1
```

또는 PowerShell에서:

```powershell
. .\server.ps1
Start-EXChat
```

기본 포트는 `8888`이다.

---

## 11. 최소 검증 시나리오

- 서버 실행 후 `http://<host>:8888/` 접속 가능
- `index.html`이 별도 파일로 서빙됨
- 브라우저 2개 이상이 같은 채팅방에 참여
- 한 브라우저 메시지가 다른 모든 브라우저에 보임
- 빈 문자열 전송이 프론트엔드에서 차단됨
- 각 메시지에 브라우저 기준 timestamp가 표시됨
- 접속 시 `C.D joined` 시스템 메시지가 표시됨
- 종료 시 `C.D left` 시스템 메시지가 표시됨
- 새로고침 시 이전 메시지는 복원되지 않음
- 한 클라이언트 연결 종료가 다른 연결을 깨지 않음
